01

Claude Code: the setup

non-negotiables
Download the desktop app first
That's home base. Grab it here →
Default to Sonnet, not Opus
Sonnet handles roughly 80% of the work at 5× fewer tokens (79.6% vs 80% on coding benchmarks, basically a tie). Only reach for Opus when you're stuck on something genuinely gnarly.
Start on the $20 plan
Don't overbuy. Upgrade the day you actually hit the ceiling, not a day before.
One folder per project
Make the folder first. Then point Claude Code at it. Don't let projects bleed into each other.
Turn on Bypass Permissions
Otherwise you'll click "accept" every 10 seconds and lose the plot. Set it once and forget it.
Build local. Push live once.
Get it right on your machine, then ship one clean push. Don't fix things in production.

Speed unlocks

small habits, huge compounding
Wispr Flow on the middle mouse button
(Or the Fn key on Mac.) Talking is faster than typing. 3 to 4 days of awkward and then you'll never type a prompt again.

Click any element on the page and ask for a change to just that piece. No more "the second card under the hero…" descriptions.
Open localhost on your iPhone via Handoff

03

Planning Mode

use it for the big stuff
Endorsed by Claude's CEO for bigger builds
You agree the plan before any code gets written. Result: faster output, fewer tokens burned, and Claude doesn't drift halfway through.
Skip it for small jobs
If the build is a quick page or a one-off prompt, planning mode is overkill. Save it for the real projects.
04

Markdown is your prompt library

the secret weapon
Keep a folder of .md files
Reusable prompts you'll run forever. Extraction, lead-magnet generator, Content Mine, SINBO 4.2, the lot. Once a prompt earns its keep, it gets a markdown file.
Drop a CLAUDE.md inside every project
Persistent context: brand, fonts, API setup, tone. Stop re-explaining yourself at the start of every session.
Don't type your brand colors. Point Claude at your live site.
Tell it to study and pull the palette and fonts directly. It's more accurate than your memory and ten times faster than digging through Figma.

06

The research move worth stealing

do this before any rebuild
Have Claude case-study the best in your space first
Before you rebuild anything, ask Claude for a deep-dive on best-in-class competitors. Structure, format, subscriber counts, what they do differently. Then ask it what to tweak in yours based on the findings. Free strategy from people who already figured it out.
